# Bamdul Reminder - PRD (Product Requirements Document)

## 1. 제품 개요

**Bamdul Reminder**는 Apple Reminder 앱의 핵심 기능을 웹에서 사용할 수 있도록 구현한 리마인더 서비스이다.

### 1.1 배경
- Apple Reminder는 iOS/macOS 전용으로, 웹 브라우저에서 접근이 제한적이다
- 크로스 플랫폼 환경에서 간편하게 사용할 수 있는 리마인더가 필요하다

### 1.2 목표
- Apple Reminder의 핵심 UX를 웹에서 충실히 재현한다
- 직관적이고 깔끔한 UI로 빠르게 리마인더를 생성/관리할 수 있도록 한다

### 1.3 대상 사용자
- Apple Reminder에 익숙한 사용자
- 웹 브라우저에서 리마인더를 관리하고 싶은 사용자

## 2. 핵심 기능

### 2.1 리마인더 (Reminder) 관리

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 생성 | 제목, 메모, 마감일시, 우선순위, 플래그를 설정하여 리마인더 생성 | P0 |
| 조회 | 목록별 또는 스마트 목록별로 리마인더 조회 | P0 |
| 수정 | 모든 필드를 인라인으로 편집 | P0 |
| 삭제 | 리마인더 삭제 | P0 |
| 완료 토글 | 체크박스로 완료/미완료 전환, 완료 시 애니메이션 | P0 |
| 하위 리마인더 | 리마인더에 하위 작업 추가 (부모-자식 계층) | P1 |

### 2.2 목록 (List) 관리

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 목록 생성 | 이름, 색상, 아이콘을 지정하여 목록 생성 | P0 |
| 목록 수정 | 이름, 색상, 아이콘 변경 | P0 |
| 목록 삭제 | 목록 삭제 시 하위 리마인더도 함께 삭제 | P0 |
| 목록 정렬 | 사이드바에서 드래그 앤 드롭으로 순서 변경 | P1 |

### 2.3 스마트 목록 (Smart Lists)

필터 조건 기반의 가상 목록:

| 스마트 목록 | 필터 조건 | 우선순위 |
|------------|----------|----------|
| 오늘 (Today) | 마감일이 오늘인 리마인더 | P0 |
| 예정 (Scheduled) | 마감일이 설정된 미완료 리마인더 | P0 |
| 전체 (All) | 모든 미완료 리마인더 | P0 |
| 플래그 지정됨 (Flagged) | 플래그가 설정된 리마인더 | P0 |
| 완료됨 (Completed) | 완료된 리마인더 | P0 |

## 3. 사용자 시나리오

### 3.1 리마인더 생성
1. 사용자가 사이드바에서 목록을 선택한다
2. 메인 영역 하단의 "+ 리마인더 추가"를 클릭한다
3. 인라인 입력 폼에 제목을 입력하고 Enter를 누른다
4. 리마인더가 목록에 즉시 추가된다
5. 필요 시 리마인더를 클릭하여 메모, 마감일, 우선순위, 플래그를 설정한다

### 3.2 리마인더 완료
1. 리마인더 항목의 체크박스를 클릭한다
2. 체크 애니메이션이 재생되고 항목이 페이드아웃된다
3. 해당 리마인더는 "완료됨" 스마트 목록으로 이동한다
4. 스마트 목록 카운트가 즉시 갱신된다

### 3.3 스마트 목록 활용
1. 사이드바 상단의 스마트 목록(오늘/예정/전체/플래그/완료)을 클릭한다
2. 필터 조건에 맞는 리마인더만 메인 영역에 표시된다
3. 메인 대시보드에서 각 스마트 목록의 카운트 요약을 확인한다

### 3.4 목록 관리
1. 사이드바 하단의 "+ 목록 추가"를 클릭한다
2. 목록 이름과 색상을 지정하여 생성한다
3. 목록을 우클릭 또는 편집 버튼으로 이름/색상/아이콘을 변경한다
4. 목록 삭제 시 확인 다이얼로그가 표시되며, 하위 리마인더도 함께 삭제된다

## 4. UI/UX 요구사항

### 4.1 디자인 방향
- Apple Reminder 앱(macOS/iPadOS)의 디자인을 최대한 유사하게 구현
- 깔끔하고 미니멀한 밝은 테마 기본 제공
- 반응형 디자인으로 모바일/태블릿/데스크탑 지원

### 4.2 레이아웃
- **2-패널 구조**: 좌측 사이드바 + 우측 메인 콘텐츠
- **사이드바**: 스마트 목록 5종(상단 고정) + 사용자 목록(하단)
- **메인 영역**: 선택된 목록의 리마인더 표시
- **대시보드**: 스마트 목록 카운트 요약 카드

```
┌──────────────────────────────────────────┐
│  Bamdul Reminder                         │
├────────────┬─────────────────────────────┤
│            │                             │
│  Sidebar   │   Main Content              │
│            │                             │
│  ┌──────┐  │   리마인더 목록 제목          │
│  │Today │  │   ─────────────────────     │
│  │Sched.│  │   ☐ 리마인더 항목 1          │
│  │All   │  │   ☐ 리마인더 항목 2          │
│  │Flag  │  │     ☐ 하위 리마인더          │
│  │Done  │  │   ☐ 리마인더 항목 3          │
│  ├──────┤  │                             │
│  │내 목록│  │   + 리마인더 추가            │
│  │ List1│  │                             │
│  │ List2│  │                             │
│  └──────┘  │                             │
│            │                             │
│ + 목록추가  │                             │
├────────────┴─────────────────────────────┤
│  Summary counts (Today: 3, Scheduled: 5) │
└──────────────────────────────────────────┘
```

### 4.3 인터랙션
- 리마인더 항목 클릭 → 인라인 편집 모드
- 체크박스 클릭 → 완료 애니메이션 (체크 후 페이드아웃)
- 사이드바 목록에 리마인더 개수 뱃지 표시
- 모바일에서 사이드바 토글 (햄버거 메뉴)

### 4.4 스마트 목록 스타일

|------------|------|--------|

| 예정 | 빨강 (#FF3B30) | 캘린더 |
| 전체 | 검정 (#1C1C1E) | 트레이 |

## 6. 우선순위 정의

| 우선순위 | 설명 |
|----------|------|
| P0 | 핵심 기능. 없으면 서비스가 성립하지 않음 |
| P1 | 중요하지만 초기 없이도 사용 가능 |
| P2 | 있으면 좋은 부가 기능 |

### P0 (Must Have)
- 리마인더 CRUD + 완료 토글
- 목록 CRUD
- 스마트 목록 5종
- 2-패널 레이아웃
- 인라인 편집

### P1 (Should Have)
- 하위 리마인더 (Subtasks)
- 드래그 앤 드롭 정렬

### P2 (Nice to Have)
- 반응형 모바일 지원
- 다크 모드
- 검색 기능
- 키보드 단축키
- 로딩/에러 상태 UI
- 애니메이션 개선
